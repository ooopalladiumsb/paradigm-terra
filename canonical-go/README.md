# `canonical` (Go parity)

Go parity implementation of the **Paradigm Terra Canonical Encoding
Specification v1.3 (SCF)** with the v0.10.0-draft domain-tag extensions
(`STATE_ROOT_V1`, `DSL_V1.2`). It mirrors the TypeScript reference
(`../canonical`, `@paradigm-terra/canonical`) and the Rust crate
(`../canonical-rs`) byte-for-byte.

Module: `github.com/paradigm-terra/canonical-go` (package `canonical`).

## Parity status

`parity_test.go` loads the committed `../canonical/vectors/golden.json`
(generated by the TypeScript reference) and independently recomputes every
vector with this package:

```
All 44 parity checks passed against TypeScript golden vectors.
```

44 field comparisons across all 17 golden vectors — integers, UTF-8 NFC, TON
address, restricted JCS, DSL v1.1/v1.2 hashes, CAL hash, stream-tree and
binary Merkle, genesis state root, framing, and the full domain-tag registry.
Identical coverage to the Rust harness.

With both **Rust** and **Go** now passing, the promotion gate in
`golden.json` (`"Promote to NORMATIVE after parity verification with Rust and
Go"`) is satisfied — the vectors can be promoted from PRE-NORMATIVE to
NORMATIVE.

## Building / testing

```sh
go test ./...
go vet ./...
```

The only external dependency is `golang.org/x/text/unicode/norm` for NFC; SHA-256,
hex, big integers, and JSON (test fixture only) come from the standard library.
Builds with `CGO_ENABLED=0` — no C toolchain required.

### Unicode version note

NFC tables come from `golang.org/x/text`. The TS reference pins Unicode 15.1
(Node 22, ICU 73+). For full conformance the `x/text` version's Unicode tables
must match the reference; all current NFC golden vectors
(`U+0065 U+0301 → U+00E9`) are stable across Unicode versions.
